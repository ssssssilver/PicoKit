# PDF 工作台产品边界

## 当前上线范围

PDF 工作台坚持“无需登录、文件不上传、浏览器本地处理”的定位，当前覆盖：

- 多 PDF 合并、页面排序、多选、旋转、删除与提取
- 图片转 PDF、PDF 转图片，以及三种模式间的内存内接力
- 批量拆分并打包 ZIP
- 裁边、A4 / Letter 页面规范化，且不栅格化原有文本与矢量内容
- 结构优化、屏幕分享、最小体积三档压缩
- 大文档窗口化渲染、可取消导入、资源压力提示
- 用户主动保存的 7 天浏览器本地草稿
- 页数与文档属性校验、交付报告

## 暂不混入当前工作台的能力

### 安全涂黑 / Redaction

只在页面上覆盖黑色矩形并不安全，原始文字和对象仍可能被复制或恢复。真正的安全涂黑需要删除内容流、注释、附件、表单值和隐藏层，并做独立验证。当前不提供容易让用户误判为“已永久删除”的伪涂黑。

### 数字签名

合法有效的 PDF 签名涉及证书、私钥保护、时间戳和验证链。把私钥导入普通网页会显著提高安全责任，也与当前无需账号的轻工具定位不符。现有工作台明确提醒：重建、拆分或压缩通常会使原签名失效。

### 密码加密 / 解密

浏览器端可通过 qpdf / WASM 等方案实现，但会新增较大的运行时和更多加密兼容测试。它应作为独立工具评估，不能用只修改权限标记的弱实现替代真正加密。

### OCR

OCR 需要语言模型、版面分析、字体回写和可搜索文本层对齐；多语言模型体积、移动端性能和准确率均超出轻量工作台范围。扫描件当前可用有损压缩，但不会宣称生成了可搜索文本。

## 产品判断

这些能力不是“少一个按钮”，而是不同的信任与验证等级。当前版本优先把页面整理、转换、拆分、规范化和压缩做完整，并对会丢失的结构进行明确说明。后续若增加上述能力，应拆成独立工具，分别设计安全边界、失败恢复和可验证的测试样本。
