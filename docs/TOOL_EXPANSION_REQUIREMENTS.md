# PicoKit 工具扩展需求与目录改版

版本：v1.1  
日期：2026-07-14  
参考对象：[Hilyx ToolBox](https://tools.hilyx.cn/)、[Toolnest](https://trytoolnest.com/)

## 1. 结论

参考站前端实际包含 35 个路由工具页，分为文本、开发、文件、计算、媒体、教育六类。公开前端包显示其技术栈为 Vue 3、Vue Router、Element Plus，并通过路由级懒加载引入 PDF、Office、图片编辑、计算器等较大的模块。

PicoKit 不照搬全部工具。新增功能必须同时满足：

1. 无需登录和付费即可完成核心任务。
2. 文件和输入默认不离开用户设备。
3. 能由 Web API、Web Worker、WebAssembly 或宽松许可证开源库完成。
4. 能形成明确的搜索需求或与现有 AI、图片、文件工具产生连续使用场景。
5. 首屏不加载工具运行库，进入对应页面后再懒加载。

按照以上原则，纳入 18 项新需求，分三批实施；7 类能力明确暂缓。

## 2. 参考站实现评估

| 模块 | 观察到的实现 | 可复用思路 | PicoKit 判断 |
|---|---|---|---|
| 文本工具 | 浏览器内字符串处理；去重、排序、统计、编码、正则、差异比较 | 实现轻、无需依赖服务器 | 合并为少量多功能工作台，避免制造大量重复页面 |
| Markdown | Markdown 转换与预览，并使用 HTML 清理库 | 本地预览安全可控 | 纳入第二批，必须使用 DOMPurify 防止预览 XSS |
| JSON / 代码格式化 | 客户端解析、格式化、高亮；代码页运行包较大 | JSON 工具需求明确 | 先做 JSON；不做多语言 IDE、混淆器和在线执行 |
| 二维码 | 浏览器生成、图片读取、样式与导出 | 无需上传，适合移动端搜索流量 | 第一批纳入 |
| PDF | PDF 解析/渲染、ZIP、Canvas 截图等前端库组合，公共包较大 | 拆分、合并、旋转、图片转 PDF 可在本地完成 | 第一批纳入，独立懒加载并设置文件上限 |
| Office | 路由包和共享依赖较大；浏览器内处理 OOXML | 表格读取和 CSV 转换可行 | 仅做 XLSX/CSV；暂缓 Word/PPT 高保真转换 |
| 图片 / 图标 | Canvas 为主的尺寸、格式、裁剪、图标导出 | 与 PicoKit 现有工具高度重合 | 补齐旋转、翻转、水印、ICO/Favicon，不另造完整编辑器 |
| GIF | 小型页面包，适合帧提取和基础合成 | 可本地处理短 GIF | 第二批纳入，限制帧数、像素和内存 |
| 音频 / 视频 | 未发现 FFmpeg 级依赖，预计依赖浏览器媒体能力 | 裁剪、静音、取帧可以轻量完成 | 音频基础编辑纳入第二批；视频仅做取帧和浏览器支持格式的轻处理 |
| 汇率 / 网络 / HTTP | 依赖外部数据源、浏览器网络权限或 CORS | 可做展示，但不符合纯本地承诺 | 暂缓 |
| 教育 / 健康 / 金融 | 计算器、词典、学习助手等，与文件工具弱相关；部分包很大 | 能增加页面数量 | 暂缓，避免品牌和维护范围失焦 |

参考站还存在产品口径不一致：首页元信息宣称“100+”，页面文案出现“26+”，实际路由为 35 个；部分首页链接、侧栏名称和真实路由也不一致。PicoKit 的工具数量只统计可访问、可完成任务的正式工具，不展示未开发完成的占位入口。

## 3. 新信息架构

工具目录固定为六个一级分类，分类表达用户任务，不按底层技术命名：

1. **AI 检测**：AI 文本、AI 图片、来源证据检查。
2. **图片处理**：抠图、水印、压缩、裁剪、格式与图标。
3. **来源与隐私**：元数据、C2PA、AI 标签、哈希校验。
4. **文件与文档**：PDF、表格、Base64、文件转换。
5. **音视频**：GIF、音频、视频轻处理。
6. **文本与开发**：文本工作台、JSON、Markdown、二维码。

目录交互要求：

- 顶部导航只保留“全部工具”、3 个高频分类、3D 和方法说明，不逐个堆放工具链接。
- 首页工具区默认展示全部已上线工具，并按分类分组。
- 分类筛选使用单选标签；搜索和分类条件可同时生效。
- 每个工具卡显示任务名称、一句话用途和运行方式。
- 移动端菜单只展示分类，不展开几十个工具。
- 未开发工具不出现在正式目录；可以在路线图文档中维护，但不得伪装成可用功能。
- 所有新增页面必须加入站内搜索、Sitemap、多语言标题和描述。

## 4. 第一批：高需求、轻依赖、优先上线

### 4.1 本地 PDF 工具箱

功能范围：

- 合并多个 PDF。
- 拆分、提取、删除、旋转、重新排序页面。
- 图片转 PDF，支持 A4、Letter、原图尺寸和边距。
- PDF 页面导出 PNG/JPEG。
- 页面缩略图预览、批量选择、撤销与下载。

技术建议：PDF.js 负责预览，pdf-lib 负责修改与导出，全部进入页面后动态加载。第一版不做 OCR、电子签名、密码破解、Office 转 PDF。

验收：100 页以内或 150 MB 以内文件能给出明确的内存风险提示；原文件不上传；导出页序与用户设置一致。

### 4.2 二维码生成与识别

功能范围：文本、网址、Wi-Fi、邮箱、电话二维码；前景/背景色、容错级别、边距、Logo；PNG/SVG 导出；上传图片识别二维码；优先尝试浏览器 BarcodeDetector，不支持时使用本地解码库。

技术建议：`qrcode`（MIT）负责生成，`jsQR`（Apache-2.0）作为识别回退。

### 4.3 文本工作台

用一个页面承载以下标签页：

- 字符、单词、行、段落和阅读时间统计。
- 按行去重、排序、删除空行、修剪空格。
- 大小写、全角半角和常用标点转换。
- URL、Base64、HTML Entity、Unicode、Hex 编解码。
- 行级和词级文本差异比较。

要求：文本仅存在组件内存；默认不写入 LocalStorage；支持复制与下载；大文本移入 Worker，避免阻塞界面。

### 4.4 JSON 工具箱

格式化、压缩、语法校验、错误行列提示、树形浏览、JSONPath 查询、JSON 与 CSV 相互转换。第一版不执行用户 JavaScript，不提供远程 URL 抓取。

### 4.5 文件校验与 Base64

- 文件计算 SHA-256、SHA-1、MD5；默认推荐 SHA-256，并标注 MD5 只用于兼容校验。
- 分块读取大文件，显示进度，支持结果复制。
- 文件转 Base64、Data URL；Base64 还原文件；图片 Data URL 预览。
- 优先使用 Web Crypto；MD5 使用体积小、许可证兼容的独立实现。

### 4.6 Favicon 与图标生成器

输入 PNG/JPEG/WebP/SVG，生成 16、32、48、180、192、512 像素图标及 manifest 片段；输出 PNG 集合和 ICO。支持安全区、背景色、圆角预览和 ZIP 下载。处理使用 Canvas/OffscreenCanvas，不上传源图。

## 5. 第二批：适合本地处理，但需要性能边界

### 5.1 图片快速编辑补全

在现有图片压缩页扩展旋转、翻转、自由裁剪、透视纠正、文字/图片水印、亮度、对比度、饱和度、灰度、EXIF 方向修正。不要新建一个重复的“完整图片编辑器”。

### 5.2 Markdown 工作台

编辑、实时预览、Markdown 与 HTML 互转、目录生成、复制和导出。任何生成 HTML 都先经过 DOMPurify 清理；不加载远程图片或脚本。

### 5.3 表格转换

XLSX/CSV/TSV 本地预览、工作表选择、CSV/JSON 相互转换、编码和分隔符选择。限制预览行数，解析放入 Worker。不得承诺复杂公式重算、宏或样式完全还原。

### 5.4 GIF 工具

GIF 拆帧、图片合成 GIF、裁剪、缩放、帧率与循环设置。设置最大像素总量和预计内存提示，低内存设备默认降低预览分辨率。

### 5.5 音频轻处理

浏览器支持格式的播放、裁剪、淡入淡出、音量调整、左右声道转单声道、波形预览和 WAV 导出。使用 Web Audio API；不承诺任意格式转换。

### 5.6 视频轻处理

视频信息检查、指定时间取帧、生成封面、旋转显示、静音和浏览器原生支持范围内的短视频裁剪。优先 WebCodecs/MediaRecorder；不可用时明确说明浏览器能力限制，不自动下载几十 MB 的 FFmpeg 包。

## 6. 第三批：低成本搜索入口

- 时间戳与时区转换。
- 常用单位转换。
- 正则表达式测试与替换。
- Cron 表达式生成和后续运行时间预览。
- 图片调色板提取、颜色格式转换、对比度检查。
- SVG 格式化、压缩预览和 PNG 导出。

第三批全部采用原生 API 或小型依赖，每项独立评估搜索量；没有明确流量信号的不进入开发排期。

## 7. 暂缓或不做

| 能力 | 原因 |
|---|---|
| 在线 HTTP 客户端、Ping、Whois、IP 查询 | 浏览器 CORS 和网络权限限制；若增加服务端代理会扩大安全与滥用面 |
| 实时汇率、历史汇率 | 需要外部 API、配额和数据新鲜度保障，不符合离线承诺 |
| Word/PPT 高保真转 PDF | 浏览器端难以保证字体、分页、动画和版式一致；通常需要 LibreOffice 等服务器能力 |
| 任意格式音视频转换 | FFmpeg WASM 下载大、启动慢、移动端内存压力高，与“轻工具”定位冲突 |
| 在线代码执行、混淆、RSA/AES 综合加密台 | 安全说明和误用成本高，且与核心图片/文件路径关联较弱 |
| 健康、贷款、投资计算器 | 结果准确性和合规说明成本高，品牌方向分散 |
| 词典、学习助手、思维导图 | 需要大型数据包或形成另一套产品体系，暂不进入 PicoKit |

## 8. 通用产品与工程要求

### 隐私边界

- 文件工具不得把 `File`、`Blob`、Canvas 像素、解析文本或处理结果发送到 PicoKit、统计或广告端点。
- 广告组件与处理组件保持独立组件树，禁止向广告组件传递文件引用或结果。
- 外部字体、字典、汇率等网络资源不得在处理过程中按用户内容动态请求。
- 每个工具页显示“本地处理”说明，但不向用户暴露具体模型文件名和下载细节。

### 性能

- 路由级动态加载；首页不得包含 PDF、表格、GIF、3D 或 AI 运行库。
- 超过 50 ms 的重复计算优先进入 Worker；提供取消操作。
- 在处理前估算解码后的像素、页数、帧数或音频时长，并给出设备风险提示。
- 移动端发生内存不足时降级预览，而不是只显示技术错误。

### 错误体验

- 错误文案说明用户可以做什么，例如“请换用 Chrome/Edge 或减少文件大小”，不显示堆栈、模型名和 CDN 地址。
- 区分不支持格式、文件损坏、内存不足、浏览器能力缺失和处理取消。
- 单项能力失败时保留仍可用的预览、元数据或其他结果。

### 开源与许可证

- 只采用允许商业网站使用的开源许可证，合并前检查许可证、NOTICE、维护状态和供应链风险。
- 推荐基础库：PDF.js（Apache-2.0）、pdf-lib（MIT）、qrcode（MIT）、jsQR（Apache-2.0）、DOMPurify（Apache-2.0/MPL-2.0）。表格与媒体库在开发前单独完成许可证和包体积复核。
- 依赖和模型统一写入 `/licenses` 页面；不要只在仓库中保留 LICENSE 文件。

## 9. 发布顺序与完成定义

建议顺序：PDF → 二维码 → 文本工作台 → JSON → 文件校验/Base64 → Favicon → 图片编辑补全 → Markdown → 表格 → GIF/音视频。

每个新工具同时满足以下条件才可加入首页正式目录：

1. 中英文界面、元数据和错误文案完整。
2. Chrome、Edge、Safari 当前主版本完成桌面验证；移动端至少验证一台 iOS 和 Android。
3. 有成功、损坏文件、超限、取消和能力缺失测试。
4. 构建产物确认没有把运行库打入首页公共包。
5. 隐私边界测试确认处理内容没有网络请求。
6. Sitemap、许可证、方法与限制说明已更新。

## 10. Toolnest 复评与轻工具增补

Toolnest 当前公开页面包含 56 个工具，采用“常用/新工具 + 六类目录 + 本地收藏”的信息架构。其图片、PDF、二维码、JSON、Base64、文本统计等主力功能与 PicoKit 已上线能力高度重合，因此不重复拆出多个只做单一格式转换的页面。PicoKit 新增需求按搜索意图、纯前端可行性和与现有工具的连续使用场景排序如下。

### P0：已于 2026-07-14 完成开发

| 工具 | 核心范围 | 前端实现 | 纳入理由 |
|---|---|---|---|
| 密码与 UUID 生成器 | 密码长度/字符集、排除易混字符、UUID v4、批量复制 | Web Crypto `getRandomValues` / `randomUUID` | 高频、零依赖、隐私价值明确 |
| 日期与时间工具 | 时间戳转换、时区换算、日期间隔、年龄计算 | `Intl.DateTimeFormat`、Temporal polyfill 仅按需加载 | 合并四类相近搜索需求，避免计算器页面泛滥 |
| 单位与比例工具 | 长度、质量、温度、面积、数据单位、图片宽高比 | 纯 TypeScript 公式 | 与图片尺寸、3D、文件大小场景相连 |
| 颜色工具箱 | 取色、图片调色板、HEX/RGB/HSL 转换、WCAG 对比度 | Canvas、EyeDropper API 可用时增强 | 与图片和网站图标流程互补 |
| 正则与 URL 工具 | 正则匹配/替换、高亮、URL 编解码、查询参数解析 | 原生 RegExp、URL、URLSearchParams | 开发者搜索需求稳定，包体积接近零 |

P0 已作为 5 个多功能页面上线：`/password-uuid-generator`、`/date-time-tools`、`/unit-ratio-converter`、`/color-tools`、`/regex-url-tools`。每个子功能仍提供可索引的标题、说明和站内搜索关键词。

### P1：已于 2026-07-14 完成开发

- SVG 工具箱（`/svg-tools`）：格式化、安全预览、尺寸检查、压缩和 PNG 导出。
- 图片头像/团队表情（`/avatar-emoji-generator`）：文字或图片、方形/圆形/圆角、安全边距和静态 PNG；GIF 动画复用现有 GIF 模块。
- 随机选择器（`/random-picker`）：名单抽取和公平分组，全部在本地运行；不加入博彩样式或积分机制。
- 本地计时器（`/timer-tools`）：番茄钟、秒表、倒计时，完成记录仅保存在当前浏览器。
- 屏幕录制（`/screen-recorder`）：使用 `getDisplayMedia` + `MediaRecorder`，只导出浏览器支持格式，由用户主动授权屏幕与麦克风。

### 暂不纳入

- BMI、卡路里、宠物年龄等健康或娱乐计算：与 PicoKit 主线弱，且健康结果需要额外准确性和免责声明。
- 小费、贷款、汇率：地区差异或实时数据依赖明显。
- MP4 转 MP3、HEIC 全兼容转换：需要较重解码/编码依赖，先不牺牲首用速度和移动端稳定性。
- 在线记事本：与“工具即用即走”的定位关联较弱；如后续开发，必须提供显式本地存储、导出和一键清空。

## 11. “我的工具”模块

目标：让无需登录的用户快速回到自己的高频工具，不引入账号、数据库或跨设备同步。

功能与交互：

1. 首页工具目录固定显示“我的工具”入口，位于“常用工具”之后，不新增顶部导航栏位。
2. 每张工具卡提供星标按钮；收藏后显示“已收藏”，再次点击取消。
3. 用户进入正式工具页时自动写入“最近使用”，最多保留 8 个，按最后访问时间排序。
4. “我的工具”先展示收藏，再补充未收藏的最近使用项，避免重复。
5. 支持只清除最近使用，或清空收藏与历史；空状态引导用户浏览常用工具。
6. 收藏与历史只写入当前浏览器的 LocalStorage，不使用 Cookie，不发送到服务端、统计或广告代码。
7. 无痕模式、禁用本地存储或用户清理浏览器数据后记录可以消失；界面不得承诺跨设备同步或永久保存。
8. 中文、英文、日间、夜间、桌面和移动端均须完整可用，星标按钮需要独立可访问名称和键盘焦点。

验收：收藏状态刷新后保留；最近访问顺序正确且最多 8 项；删除未知或已下线工具记录；LocalStorage 损坏或不可用时页面仍正常工作；收藏按钮与工具详情链接不形成嵌套交互元素。
